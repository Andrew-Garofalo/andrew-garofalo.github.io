Use Cases

Create Driver Account


Basic Course:
When the Driver clicks the Create Account button, the system shows the Create Account page. The system generates and displays a random ID and the Driver enters a password and clicks the Submit button. The system makes sure password is valid and creates the account. The system displays a Confirmation page.

Alternative Course:
Driver enters invalid password: System prompts the Driver on the Create Account page to enter a valid password. The Driver then types in new password and the system verifies that it is valid. No password entered by Driver: System displays error message on the Create Account page screen that Driver password field must not be blank. The Driver then types in a password and the system verifies that it is valid.

Create Form

Base Course:
Admin selects either Selected Combo Box or Selected Text Box and clicks the Add Button. If Selected Text Box is chosen, system displays additional option of specifying a Regular Expression to restrict Admin’s input. The system prompts the Admin to drag and drop the components to the Form. The Admin marks component as being a required field, the system displays an arrow to show this visually. Admin clicks the Save Form Button and system saves the Form to Admin Account.

Alternative Course:
Form Creator not logged in: System displays Login Page. Driver is logged in: System displays Home page with an error message Regular Expression Incorrect: System displays error message

Fill Out Form Use Case


Basic Course:
System displays CDL Form Page. Driver fills out the forms with their PHI, age, and city. System displays certain questions depending on the Driver responses. Driver clicks Submit Button and the system displays the Results Page.

Alternative Courses:
Driver Saves and Exits Form: Driver clicks the Save Button on the form. System saves all of the information to the Database. Driver closes the application and system logs the Driver out. Invalid Data Entered: System displays error message for Driver to enter correct information. Incomplete Form Submitted: System alerts Driver to fill out all needed information.

Results

Basic Course:
Driver clicks Saved Forms button, system displays Saved Forms page. System displays all previously created Questionnaire forms. Driver clicks the Download Medical Examination Report Form link then clicks the Done button and the system displays the Homepage.

Alternative Course:
Waiver Forms are needed: System displays links for applicable Waiver Forms and prompts Driver to download the Waiver Forms and Medical Examination Report. User closes the page and reopens it: System invokes Login use case. Driver re-enters their ID and password.

Class Diagram

Domain Model

High Level Requirements

  1. The system shall allow the Admin the ability to create forms.
  2. The system shall allow the Admin the ability to edit forms.
  3. The system shall allow the Admin and Driver to create accounts.
  4. The system shall allow the Driver to fill out the CDL Questionnaire.
  5. The system shall allow the Driver to print off their forms after filling out the CDL Questionnaire.
  6. The system shall allow the Admin to query the database containing the Driver Account Information.
  7. The system shall allow the Admin to back up the database containing the Driver Account Information.
  8. The system shall allow the Driver to keep up with current CDL Licensing news.
  9. The system shall allow the Driver to save their CDL Questionnaire and resume it later.
  10. The system shall provide the Driver with information regarding any CDL Licensing restrictions that may apply, based off of their CDL Questionnaire responses.

Project Glossary

  1. Driver: The truck drivers who will be filling out the CDL Prescreener questionnaire.
  2. Admin: The users who will be developing/monitoring the application.
  3. Database: Stores the data used by the application.
  4. Driver Account: Contains all the Protected Health Information (PHI), age, and driver’s city that is entered in the application. Will NOT have any Personally Identifiable Information (eg. Name, Full Address, SSN, Date of Birth, phone number, etc).
  5. Driver Account Information: Contains all of Driver’s PHI (from the questionnaire), age and city.
  6. Protected Health Information (PHI): is any information about health status, provision of health care, or payment for health care that is created or collected by a Covered Entity (or a Business Associate of a Covered Entity), and can be linked to a specific individual.
  7. Admin Account: Allows Admin to access protected areas of the application.
  8. Form Builder: A GUI that Admins can use to build the questionnaire.
  9. CDL Questionnaire: The form that the Driver will be filling out with their health information.
  10. CDL Forms: The forms that might need to be printed out by the Driver at the end of the questionnaire process.
  11. Medical Examination Report Form: The form that will be populated with the Driver’s answers from the questionnaire, that will be printed out by the Driver.
